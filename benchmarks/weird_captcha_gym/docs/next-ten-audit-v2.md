# Next-ten benchmark audit v2

This cohort was audited against the interaction-puzzle field notes rather than
the weaker condition that a scripted green path can submit. The binding test is
whether the task creates real computer-use debt—motion, time, closed-loop
control, multi-view geometry, physical manipulation, or delayed causality—and
whether a normal human action can be rejected only because of browser/VNC event
density.

## Binding acceptance gates

1. A correct result cannot be recovered as a static OCR, classification, or
   logical-QA answer.
2. Rendered geometry, hit geometry, client state, live server replay, and the
   exported verifier implement the same contract.
3. Event sampling density is not correctness. Sparse VNC pointer delivery is
   accepted whenever the semantic continuous-state condition is met.
4. Deliberate negative/recovery coverage runs on a challenge that is discarded.
   A perfect accepted transcript is not forced to contain mistakes.
5. Initial, active, local failure/recovery, solved, and explicit terminal states
   are visually inspectable.
6. At least three fresh browser cohorts pass server grade, direct grade, task
   verification, and console checks.
7. Human VNC review remains pending. Automation cannot promote any task to the
   verified split or establish model difficulty.

## Verification record

- Canonical evidence: `evidence/next_ten_audit_v2/` (63 PNGs plus the exact
  ten-mechanic replay summary).
- Three complete post-repair browser cohorts passed: 30/30 accepted runs. Each
  cohort also exercised a deliberate failure/fresh challenge, ordinary
  mouse/keyboard interaction, live server grade, direct grader, exported task
  verifier, and browser console/page-error collection.
- Eighty additional generator seeds per mechanic passed construction and
  structural-variation checks (800 generated tasks). Wrong Number covered all
  five carrier slots; Bomb Manual covered all seven wire positions.
- Forklift additionally covered all 128 generated layout/transform/palette
  combinations over a 1,000-seed sweep. All eight physical topologies were
  played through the browser and independently accepted by server, direct
  grader, and exported verifier.
- Accepted results for all ten mechanics were separately mutated. Task identity,
  transcript ordering, and public challenge tampering were rejected. Semantic
  mutations to signal residuals, acetate poses, vehicle physics, hold movement,
  die orientation, delayed queue state, panorama geometry, socket acceptance,
  drawing continuity, and market orders were also rejected.
- The strict quality audit reports only the intentional
  `prototype_visual_candidate / local_verification_pending_vnc_human_evidence`
  status blocker for each task. None was promoted.
- Canonical repository scope: `python -m pytest tests -q` completed with
  **207 passed, 21 skipped**. The two emitted warnings are upstream
  deprecation notices, not puzzle failures.

## Honest remaining boundary

These results establish implementation coherence, not benchmark value. No
computer-use-agent evaluation has yet shown discriminating power, and this
cohort remains pending real VNC human calibration. Insider Trading still loads
future quotes into browser JavaScript state even though they are not rendered;
it is appropriate for screenshot/mouse/keyboard agents but not for an agent
allowed to inspect page or network internals.
