## Non-negotiable gates

- No accepted trajectory contains a collision, reset, intentional wrong probe,
  strike, or other mistake unless that mistake is intrinsically required by the
  stated mechanic.
- Visible geometry, pointer geometry, local simulation, live grading, and the
  exported verifier describe the same generated world.
- Each changed generator is exercised across enough fresh seeds to demonstrate
  structural, not palette-only, variation and reachable instances.
- Each mechanic gets a deliberate invalid attempt, fresh failure state, clean
  ordinary-input browser solve, live server pass, direct grader pass, exported
  verifier pass, zero page/console errors, and initial/active/solved/verdict
  screenshots.
- Solution films are captured only after task, generator, frontend, grader,
  verifier, and solver hashes are frozen. Capture pacing may improve legibility;
  it may not change task difficulty.
- These ten remain `pending` after automation. Only the user can mark them
  `looks_good`, `approved`, or `revision_requested`.

## Verified outcome

The contract above was completed without promoting any review state. The six
retained spatial tasks were revalidated, and the four weak tasks were changed
as follows:

- Code-to-Diagram now has nine generated nodes, two decisions, four
  branch-covering transient probes, 28 debugger steps, and ten physical cords.
- Modal Terminal Escape now requires visiting three read-only reference
  buffers, repairing six distinct manifest lines through ordinary Vim input,
  writing and quitting, and unwinding pager, job, SSH, and multiplexer layers
  in a generated order.
- Fake Desktop now requires two ordered seal transfers, two independently
  moved windows, one closed blocker, and three distinct remote-cursor mappings.
- Impossible Ecology no longer asks for a classified culprit. It is a
  continuous coupled-control task in which three global fields move all five
  inertial organisms and the operator must shepherd every organism around a
  solid obstacle into its matching sanctuary.

The canonical browser run and a second independent seed-prefix run both passed
all ten mechanics through the live server, direct grader, and exported task
verifier. Accepted trajectories report zero collisions/resets where those
budgets apply. A dedicated generator/shortcut test then exercised 48 seeds per
mechanic (480 generated instances), required 48 non-cosmetic structural
signatures for each mechanic, and proved that empty terminal-state claims and
cross-task replay are rejected.

That identity test exposed a real browser-boundary omission in the three newly
strict graders: Code-to-Diagram, Vim, and Fake Desktop originally omitted
`task_id` from their browser submissions. The browser payloads now carry it,
and the graders check the result, public state, and private contract before
replay. Film capture also exposed a LIDAR solver timing assumption under video
encoding; short feedback-controlled ordinary key holds replaced sampled
sign-crossing waits. Three additional fresh facilities then passed with about
1,630 replayed movement ticks each and zero accepted collisions. Neither fix
changes a task or loosens acceptance.

Canonical evidence is in `evidence/pending_next_ten_v3/`. Ten MP4/WebM solution
pairs are in `evidence/pending_next_ten_v3/solution_videos/`; all are 1280×720,
have zero browser errors, and pass live/direct/exported verification. Their
manifest proves that the frozen task-facing hashes did not change during
capture. Review status remains `pending` for all ten.
